TP3 : Manipulation des principales commandes GIT de gestion de modifications de fichiers.

Le but de ce TP est de comprendre et d'utiliser les principales commandes de GIT, et gérer la "staging area" (index)

On utilisera le dépôt du TP2 pour réaliser cet exercice

Modification d'un fichier

  • Dans la branche principale, modifier un seul fichier. Utiliser "git status" pour visualiser les modifications
  • Ajouter seulement ce fichier à l'index
  • Commiter cette modification

Modification de plusieurs fichiers

  • Dans la branche principale, modifier au moins 2 fichiers
  • Utiliser la commande git add pour n'ajouter qu'un seul des fichiers à l'index
  • Commiter cette modification
  • Ajouter ensuite le 2eme fichier à l'index
  • Commiter cette seconde modification

Annuler un commit

  • Modifier un fichier, puis ajouter et commiter
  • Annuler ce commit (revenir sur la version précédente) avec la commande
$ git reset --soft HEAD~1

Ou bien

$ git reset --soft HEAD^

HEAD^ signifie le commit parent de la version courante (HEAD)

HEAD^^ signifie 2 commits en arrière depuis la version courante

HEAD^^^ signifie 3 commits en arrière depuis la version courante

etc...

Ou encore

$ git reset --soft HEAD~1
$ git reset --soft HEAD~2
$ git reset --soft HEAD~3

Ou encore

$ git reset --soft <sha1>

Reset Hard

Recommencer la section "annuler un commit" avec la commande reset --hard

Changement de chemins

  • Modifier le nom d'un fichier (README.md => README.txt)
  • Utiliser git status pour voir les changements
  • Ajouter et commiter cette modification

Suppression de fichier

  • Supprimer un fichier
  • Utiliser git status pour voir les changements
  • Commiter cette modification

Historique des commits

  • Afficher l'historique des commits en affichant chaque commit sur une seule ligne
  • Créer un alias git qui permet d'afficher l'historique des commits de manière condensée
  • Utiliser cette alias
