TP 9 : Le cherry-pick

Le but de ce TP est de comprendre le cherry-pick

Nouvelle branche

  • Créer une nouvelle branche depuis la branche master
  • Y faire 3 commits
  • Repérer le commit du "milieu" et son hash

  • Se placer à nouveau sur master et utiliser la commande suivante pour répercuter le cherry pick sur la branche master :

    git cherry-pick <commit hash>

Conflits

Des conflits peuvent aussi apparaitre lors d'un cherry-pick. Dans ce cas, résoudre le conflit, puis :

git add fichier-en-conflit
git cherry-pick --continue
