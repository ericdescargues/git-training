TP2 : Création et initialisation d'un dépôt. Utilisation de l'outil de visualisation Gitk

Le but de ce TP est de découvrir la notion de dépôt, et de le parcourir via l'outil graphique gitk.

Il est aussi question de manipuler l'outil git gui pour faire les mêmes opérations qu'en ligne de commande.

Création d'un dépôt local depuis un dépôt distant

Pour récupérer un dépôt distant et son contenu, et créer un dépôt local, on utilise la commande git clone

$ git clone https://github.com/ericdescargues/training-git.git

Inspecter le dossier créé dans votre dossier courant.

Inspecter le sous dossier .git et son contenu.

Afficher et inspecter l'historique avec

$ gitk

Création d'un dépôt local

  • Créer un nouveau répertoire dans le dossier utilisateur, y ouvrir un terminal
  • Initialiser un dépôt avec la commande git init
  • Créer plusieurs fichiers texte, avec du contenu

Pour créer rapidement des fichiers avec du contenu :

$ echo "Some text" > mon-fichier.txt

Création du premier commit

On ajoute les fichiers à l'index, puis on "commit"

$ git add --all
$ git commit -m "Mon premier commit"

Lire l'historique des commits avec la commande

$ git log

Visualisation du dépôt avec gitk

  • Dans le répertoire du dépôt, lancer la commande gitk.
  • Parcourir le dépôt avec l'interface graphique.
  • Repérer les actions clés de l'interface.

Découverte de git gui

Git gui est une interface graphique permettant de faire les mêmes opérations qu'en ligne de commande.

Pour lancer git gui, ouvrez simplement un shell ou GitBash à la racine de votre dépôt et lancer la commande git gui

Répéter les opérations de :

  • Création d'un deuxième fichier txt
$ echo "Some text bis" > mon-fichier-bis.txt
  • Ajouter ce fichier dans l'index
  • Commit du fichier.
